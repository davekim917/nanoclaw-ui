## Design Context

### Users
Solo developers and small dev teams (2-10) managing NanoClaw instances for AI-assisted automation. They are technically proficient, comfortable with APIs, and use the UI as a control surface for chat, workflows, skills, and session history. Context: daily driver tool, often alongside an IDE or terminal.

### Brand Personality
**Sharp, capable, quiet.** Professional tool that earns trust through restraint. No flash without function. The interface should feel like a precision instrument — responsive, decisive, and never in the way.

### Brand Identity
- **Icon:** Abstract pincer (`ICON_PATHS.pincer`) — two converging angular arms suggesting a claw grip. Distinctive, not generic. Used for brand mark, assistant avatars, loading states, and empty states.
- **Favicon:** Pincer motif on navy `#0f172a` background, teal `#2dd4bf` strokes.
- **Wordmark:** "NanoClaw" in Inter 700, `letter-spacing: -0.02em`.

### Aesthetic Direction
- **Visual tone:** Dark-first, minimal, confident. Navy surfaces with flat teal accent. Premium without being precious.
- **Flat over gradient:** All buttons, brand marks, and interactive elements use flat `var(--color-accent)` — never gradients. Hover states use `var(--color-accent-hover)`. No glow shadows on decorative elements.
- **References:** Town.com (conversational UX, real-time feel), ChatGPT/Claude.ai (chat-centric layout, streaming UX, input patterns), Linear/Raycast (keyboard-driven polish, fast transitions).
- **Anti-references:** Enterprise bloat (Jira-style density), generic template kits (Bootstrap/Tailwind starters), retro terminal aesthetic (green-on-black), playful/whimsical UI (cartoon shapes, emoji-heavy), AI slop (teal gradients + glow shadows + bolt icons).
- **Theme:** Dark mode default (`#0c1222` navy base, `#2dd4bf` teal accent). Light mode available. Both use the same teal brand accent.

### Emotional Goals
- **Confidence & control** — the user feels in command of a powerful system. Everything is predictable and responsive.
- **Speed & efficiency** — no wasted space, no unnecessary steps. The interface gets out of the way and lets the user work.

### Design Principles

1. **Quiet confidence** — Every element earns its place. No decorative noise: no gradients on buttons, no glow shadows, no pulse animations. Flat colors, clean borders, purposeful whitespace. If a surface doesn't help the user act, remove it.

2. **Conversation-first** — Chat is the primary interaction. Design decisions should optimize for reading, scanning, and composing messages. Other pages (skills, workflows, sessions) are supporting surfaces.

3. **Actionable empty states** — Never show passive "Nothing here" text. Every empty state must include: an icon, a title, an explanation of what will appear, and a CTA that guides the user toward the next action. Loading states use skeleton shimmers, not text spinners.

4. **Systematic, not generic** — All colors through tokens. No hardcoded hex or rgba in component CSS. Use the token system (`tokens.css`) consistently. Spacing on 4/8px grid, stroke-based 24x24 icons, Inter/JetBrains Mono type pairing. Custom-styled controls over native elements (e.g., custom dropdown over `<select>`). The result should feel bespoke, not templated.

5. **Responsive precision** — Transitions are fast (120-200ms), interactions feel immediate, and the layout adapts cleanly between desktop sidebar and mobile drawer. Touch targets are 44px minimum. Safe-area insets for mobile. No janky reflows.

6. **Dark-first, light-ready** — Design for dark mode first; light mode inherits the same spatial logic with adjusted surface/text colors. Both themes must define all custom tokens used by components.

### Tech Constraints
- **Lit 3.x web components** — CSS-in-JS via `css` tagged template literals in each component's `static styles`. Shadow DOM isolates styles; rely on CSS custom properties from `tokens.css` for theming. Global CSS classes (e.g., `.skeleton`) don't penetrate shadow DOM — duplicate keyframes/classes in each component that needs them.
- **No CSS framework** — All styling is hand-authored using design tokens. No Tailwind, no component library.
- **DOMPurify** — All user/assistant HTML is sanitized. Security is non-negotiable.
- **Inter + JetBrains Mono** — Font pairing via Google Fonts. Inter for UI, JetBrains Mono for code/tokens.
- **SVG stroke icons** — Shared `ICON_PATHS` object in `src/utils/icons.ts` with 24x24 viewbox path data. Stroke-based, not filled. Brand icon is `pincer`, not `bolt`.
- **Destructive actions** — Require confirmation (two-click pattern with auto-dismiss timeout).
